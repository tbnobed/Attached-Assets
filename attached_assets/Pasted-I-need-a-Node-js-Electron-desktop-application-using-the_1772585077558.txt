I need a Node.js/Electron desktop application using the Zoom Video SDK that captures each participant's video and audio as a clean, isolated feed — similar to Zoom ISO but with custom routing capabilities. This is for a professional broadcast television production environment.

Core Requirements

Session Management

Connect to a Zoom Video SDK session using JWT authentication (SDK Key + Secret)
Display a participant roster showing who is connected
Handle participant join/leave events in real time
Support up to 25 participants simultaneously


Isolated Video Capture (Per Participant)

Capture each participant's raw video frame stream independently
No UI overlays, name tags, reactions, or Zoom chrome on any feed
Each feed should be full frame — no cropping or compositing
Target resolution: 1080p (with 720p fallback)
Target frame rate: 30fps


Isolated Audio Capture (Per Participant)

Capture each participant's raw audio stream independently
Clean, unprocessed audio — no Zoom noise suppression applied if possible
Separate audio track per participant, not a mixed output


Output Options

NDI Output: Push each participant as a named NDI source onto the local network (e.g., "Participant_JohnDoe_Video", "Participant_JohnDoe_Audio") so OBS or a hardware switcher can pick them up
Local ISO Recording: Record each participant's video + audio as a separate MP4 file to a user-defined output directory, named by participant display name and timestamp
Both NDI output and local recording should be able to run simultaneously


Control Dashboard (Simple UI)

Show a live thumbnail preview of each participant feed
Toggle ISO recording on/off per participant or all at once
Toggle NDI output on/off per participant or all at once
Show recording status, duration, and file size per participant
Show session connection status and participant count
Allow setting the output directory for ISO recordings




Tech Stack

Runtime: Node.js with Electron (for desktop deployment on Windows/Mac)
Zoom SDK: @zoom/videosdk (latest)
NDI: Use the grandiose npm package for NDI output, or node-ndi as an alternative
Video Encoding: fluent-ffmpeg for writing MP4 ISO files from raw frame/audio buffers
UI: React (inside Electron renderer process) with Tailwind CSS — keep it minimal and functional, broadcast-style dark UI
Auth: JWT token generation from SDK Key + Secret stored in a local .env file

Key Technical Notes

Use Zoom Video SDK's stream.captureVideoFrame() or equivalent to pull raw frames per participant — do NOT use the built-in renderer which adds overlays
Audio should be captured via the SDK's audio stream API, buffered, and piped to both FFmpeg (for recording) and NDI audio send
NDI sources should be created on session join and destroyed on participant leave
FFmpeg should receive raw RGBA/YUV frame buffers and PCM audio piped via stdin — avoid writing intermediate temp files for performance
Electron's main process should handle all heavy stream processing; the renderer (UI) should only receive status updates via IPC — keep the UI lightweight
